请帮我创建一个纯前端、无后端的军事装备展示网站。网站只需要两个核心功能：产品展示 + 联系厂家。具体要求如下： 
  
 ## 技术栈 
 - 纯 HTML/CSS/JS，不使用任何框架或第三方库（但允许调用外部 API）。 
 - 全静态，可部署在任何对象存储或 CDN 上。 
 - 所有交互必须在前端完成，不留任何用户数据在后端。 
  
 ## 功能需求 
 1. **产品展示**：页面展示至少 4 个军事装备产品（示例数据见下文），每个产品包含图片（可用占位图）、名称、简短描述。 
 2. **点击联系**： 
    - 用户点击任意产品的“联系”按钮后，该产品卡片立即从页面中隐藏（视觉上消失）。 
    - 同时弹出一个模态窗口（居中、半透明背景），窗口内显示： 
         - “请发送邮件至：” + 一个动态生成的临时邮箱地址。 
         - “邮件标题请注明：产品名称 + 您的需求。” 
         - “厂家将在 24 小时内回复您。” 
         - 窗口底部有一个 3 秒倒计时提示，倒计时结束后窗口自动关闭。 
    - 临时邮箱地址通过调用 Guerrilla Mail 的公开 API 实时获取：` `https://api.guerrillamail.com/ajax.php?f=get_email_address` `。如果该 API 无法访问，则降级显示一个固定的备用邮箱（如 `contact@example.com`），并提示“临时邮箱服务不可用，请使用备用邮箱”。 
    - 对于同一个产品，同一个浏览器中只能联系一次（刷新页面后该产品依然隐藏，使用 localStorage 记录已联系的产品 ID）。 
 3. **消失逻辑**：产品隐藏后，刷新页面应保持隐藏状态（通过 localStorage 读取）。如果用户清除了浏览器数据，产品会重新出现（这是可接受的）。 
 4. **全球访问友好**：页面需自适应手机/电脑，字体、间距合理。所有资源尽量使用 CDN 或本地，确保在全球大部分地区快速加载。 
  
 ## 示例产品数据 
 | 产品名称 | 描述 | 
 |---------|------| 
 | 天眼-3 侦察无人机 | 长航时、隐身设计，适用于边境巡逻 | 
 | 玄武-2 装甲车 | 4x4 全地形，防雷等级达到 STANAG 4569 标准 | 
 | 闪电-7 防空系统 | 近程防空导弹系统，可拦截无人机和巡航导弹 | 
 | 幽灵-1 单兵外骨骼 | 增强士兵负重能力，内置 AI 辅助瞄准 | 
  
 图片可用 Font Awesome 图标或灰色块代替，不需要真实图片。 
  
 ## 样式要求 
 - 简洁、军事风格（深色背景、墨绿/灰色调、硬朗边框）。 
 - 产品卡片采用网格布局（每行 2 个，手机上自动变成 1 个）。 
 - 弹窗背景半透明黑色，内容白色卡片，带阴影。 
  
 ## 安全性考虑 
 - 不收集任何用户个人信息（不要求填手机、邮箱等）。 
 - 不使用任何后端服务器，无数据库。 
 - 厂家联系方式完全不暴露在代码中（临时邮箱由 API 动态生成，即使被抓包也无法长期使用）。 
 - 使用 HTTPS 调用 API（Guerrilla Mail 支持 HTTPS）。 
  
 ## 部署说明 
 - 生成一个单独的 `index.html` 文件，包含所有样式、脚本和 HTML。 
 - 文件应自包含，可直接在浏览器打开运行，也可上传到任何静态托管（如 Cloudflare Pages、GitHub Pages、阿里云 OSS 等）。 
 - 请添加必要的注释，便于后续修改。 
  
 ## 额外提醒 
 - 在页面底部添加法律声明：“本网站仅展示公开信息，所有产品仅供参考。联系前请确认具备合法资质。” 
 - 如果 Guerrilla Mail API 被墙或超时，弹窗中应给出备用邮箱，并记录错误到控制台，但不要中断用户体验。
